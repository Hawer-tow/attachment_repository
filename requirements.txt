## API & Integration Requirements

### Authentication
- JWT Tokens: Laravel Sanctum
- Authorization Header: Bearer token required for all endpoints

### Response Format
- JSON: All API responses in JSON format
- HTTP Status Codes: Standard REST conventions

## Performance & Scalability

### Caching
- Config Caching: php artisan config:cache
- Route Caching: php artisan route:cache
- View Caching: php artisan view:cache

### Optimization
- Database Indexes: On commonly queried columns
- Pessimistic Locking: For booking conflict prevention
- Query Optimization: N+1 query prevention

### Concurrency Handling
- Database Isolation Level: READ COMMITTED or higher
- Transactions: For multi-step operations

---

## Known Limitations & Constraints

- Single Property: Designed for one hotel property only
- Basic Yield Management: Manual rate overrides, not advanced algorithms
- Background Queuing Required: Email processing requires queue worker
- WebSocket Fallback: Uses polling if real-time connection unavailable
- Tape Chart: Default 14-day view; performance may degrade with very long date ranges
- Frontend Role Restrictions: Admin features not yet role-restricted in UI

---
